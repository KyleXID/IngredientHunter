01 — 개요

서비스 정의

사용자가 음료 뒷면 성분표를 촬영하면 성분을 인식해 리스크를 분석하고 리포트 카드로 제시함. 화장품 리뷰 서비스 "화해"의 건강식품 버전에 해당함. 다만 제품의 핵심 자산은 UI가 아니라 교차검증된 성분 리스크 지식베이스임. 근거 레벨(동물실험·in-vitro·RCT)과 출처, "컨센서스 vs 논쟁"을 구분해 담으며, 여기서 기존 서비스 대비 근거·투명성이라는 차별점이 발생함.

02 — 사용자 흐름

사용자 흐름

식품·화장품 스캐너 Yuka(사용자 8600만)가 가장 근접한 레퍼런스임. 아래 흐름을 참고함.

  1. 진입 — 스캔이 첫 화면

    온보딩 직후 강제 가입은 이탈 요인으로 지적됨. 가입은 최대한 뒤로 미루고 스캔을 우선 노출함.

  2. 촬영 — 성분표 인식

    Yuka는 바코드 기반이나, 제품 데이터베이스가 없는 초기에는 성분표 사진을 직접 인식함. 카메라 권한은 스캔 직전에만 요청함.

  3. 분석 — 대기 화면

    비전이 성분을 추출·매칭하는 동안, 근거를 대조 중임을 드러내는 신뢰감 있는 대기 화면이 필요함.

  4. 결과 — 등급 우선, 상세 후행

    종합 등급을 상단에 크게 배치한 뒤, 좋은 점 / 나쁜 점을 분리하고 성분은 접이식 섹션으로 제공함.

  5. 히스토리 — 스캔 이력

    지난 스캔을 훑어보는 리스트. 스와이프 삭제 등 익숙한 제스처를 적용함.

참고할 UX 패턴

  • 초록–주황–빨강 3색 등급 코딩
  • 당·나트륨은 수치 대신 초록→빨강 스펙트럼 바
  • "주의 성분 N개" 형태의 요약 칩
  • 접이식 섹션으로 정보 밀도 조절

차별 지점

  • 바코드 미사용 — 성분표 사진을 인식함. 바코드는 제품 DB 확보 후 M4에서 검토함
  • 정밀 점수(0–100) 미도입 — 스코어링 모델 부재로 MVP는 성분별 리스크 배지 + 요약으로 시작함. 합성 점수는 데이터 확보 이후 도입함
  • 대안 제품 추천도 제품 DB 확보 이후 단계임

04 — 톤앤무드

비주얼 톤앤무드

비주얼 방향은 디자이너 주도임. 아래 내용은 확정안이 아니라 공유 앵커이며, 색·타이포·비주얼의 최종 결정은 디자이너 무드보드와 Figma에서 이뤄짐. 기획이 지키는 원칙은 하나 — 근거·출처의 투명성임.

디자이너 무드보드에서 읽은 방향

키워드는 웰니스 × 헬스테크임. 라임·딥블루·파스텔·소프트 뉴트럴을 오가는 앱 팔레트, 헬스 스코어·레이팅 대시보드, 미스티한 웰니스 사진, 세리프 에디토리얼이 공존함. 브랜드 톤 참고는 Well Mate · wellness.mate(웰니스 매거진 톤 — 따뜻·차분·라이프스타일)임.

wellness × health-tech 헬스 스코어 대시보드 라이프스타일 사진 editorial 근거·투명성

보조 앵커 — AG1 "life + science"

근거 기반·백과사전적·미니멀·모듈러. 과도한 장식 없이 정보를 품격 있게 전달하는 결로, 무드보드의 웰니스 톤과 "신뢰·근거" 축에서 만남.

06 — 디자인 원칙

준수 · 지양 사항

✓ 준수

  • 근거 레벨·출처를 항상 함께 노출함
  • 색은 의미(양호·주의·위험)로만, 3색으로 고정함
  • 중립·차분한 근거 기반 카피를 사용함
  • 정보 밀도는 접이식·시각 척도로 조절함

✕ 지양

  • 근거 없는 단정("몸에 나쁨" 식 표현)
  • 공포 유발·과장 그래픽
  • 데이터 없는 가짜 정밀 점수(0–100)
  • 브랜드 액센트를 여러 색으로 분산함

08 — 협의

협의가 필요한 사항

  • 비주얼 톤은 디자이너 무드보드 주도로 진행함 — 기획이 지키는 축은 근거·출처 투명성 하나임
  • 서비스명은 가칭임. 네이밍·워드마크 방향을 함께 확정하고자 함
  • 결과 화면 점수는 등급+배지로 시작함 — 무드보드의 숫자 스코어는 스코어링 데이터 확보 후 도입함
  • 선착수 화면 우선순위 — 결과 카드를 우선 제안함
