06 · 핵심 개념

어노테이션(@) = 파이썬 데코레이터의 사촌

@무언가는 클래스·함수에 "표식"을 붙여, 프레임워크가 그걸 보고 특별히 취급하게 하는 것임. 실행 코드가 아니라 "메타 정보".

  • @RestController → "이 클래스는 API 뷰들의 모음" (DRF @api_view/APIView 느낌)
  • @Service → "이건 로직 담당 부품, 등록해두고 필요하면 꽂아줘"
  • @PostMapping("/analyze") → "POST /analyze 오면 이 함수" (urls.py의 path)
  • Django 데코레이터와 달리 런타임에 프레임워크가 스캔해서 배선함.
07 · 컨트롤러

AnalyzeController — 한 줄씩

@RestController                 // ① API 컨트롤러
@RequestMapping("/api")         // ② 공통 접두 URL
class AnalyzeController(
    private val analyzeService: AnalyzeService  // ③ 주입
) {
    @PostMapping("/analyze")     // ④ POST /api/analyze
    fun analyze(@RequestBody req: AnalyzeRequest): Any =  // ⑤
        analyzeService.analyze(req.imageBase64, req.mediaType)
}
  • ①②④ = Django urls.py의 path("api/analyze", …) + DRF @api_view(["POST"])를 어노테이션으로 붙인 것.
  • ③ 생성자에 서비스를 선언만 하면 스프링이 알아서 넣어줌(다음 장).
  • ⑤ @RequestBody = 요청 JSON을 객체로 자동 변환(DRF Serializer/ request.data).

09 · 요청 파싱

data class + @RequestBody = Serializer

Kotlin
// JSON {imageBase64, mediaType} 을
// 이 타입 객체로 자동 변환
data class AnalyzeRequest(
    val imageBase64: String? = null,
    val mediaType: String? = null,
)
Python · DRF
class AnalyzeRequest(serializers.Serializer):
    imageBase64 = serializers.CharField(
        required=False, allow_null=True)
    mediaType = serializers.CharField(
        required=False, allow_null=True)
  • data class = 값만 담는 가벼운 클래스(파이썬 @dataclass/ DRF Serializer와 유사).
  • 필드에 타입이 박혀 있어, 잘못된 JSON이면 컴파일·요청 단계에서 걸림(타입 안전).
10 · 서비스

@Service + @Value(설정 주입)

@Service   // "로직 부품"으로 등록 → 컨트롤러에 주입 가능
class AnalyzeService(
    @Value("\${anthropic.api-key:}") private val apiKey: String,   // 설정값 주입
    @Value("\${anthropic.model:claude-sonnet-5}") private val model: String,
)
  • @Value("\${anthropic.api-key:}") = properties/환경변수의 값을 생성 시 자동으로 꽂음.
  • Django라면 settings.ANTHROPIC_API_KEY를 서비스 안에서 읽는 것과 같음 — 다만 여기선 주입받음.
11 · import

왜 이렇게 import가 많나

import …web.bind.annotation.*   // @RestController 등
import …stereotype.Service       // @Service
import …annotation.Value         // @Value
import …core.io.ClassPathResource// 리소스 읽기
import tools.jackson…JsonMapper  // JSON
import java.net.http.HttpClient  // HTTP(표준)
  • 파이썬 from … import …와 개념 동일 — 쓰는 것만 명시적으로 가져옴.
  • 차이: 패키지 경로가 도메인 역순으로 길고, 안 쓰는 import는 경고가 남.
  • java.net.http·java.*는 JDK 표준이라 설치 불필요(파이썬 requests는 pip 설치 vs 표준 http.client).

14 · 로직 & 코틀린 특징

analyze() 흐름 + null 안전

fun analyze(imageBase64: String?, …): Any {
  // 키 없으면 데모로 폴백
  if (apiKey.isBlank() || imageBase64.isNullOrBlank())
      return json.readValue(DEMO, Map::class.java)
  … Claude 호출 …
  val text = (body["content"] as? List<*>)
      ?.firstOrNull { it["type"] == "text" }
      ?.get("text") as? String
      ?: return mapOf("error" to "empty")
  return json.readValue(stripFence(text), Map::class.java)
}
  • String?의 ? = "null일 수 있음". 파이썬엔 없는 컴파일러 null 검사.
  • ?. null이면 건너뜀 · ?: null이면 기본값(엘비스) · as? 안전 캐스팅.
  • 덕분에 NoneType 에러가 런타임에 터지는 일을 코틀린은 컴파일 때 막음.
